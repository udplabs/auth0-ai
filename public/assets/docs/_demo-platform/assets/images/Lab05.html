Lab 5 Objective

Integrate a functioning FGA client into the demo application. This will enable you to verify that Aiya can correctly return sensitive account information with valid results.

Scenario

As a customer of The bAInk, it is expected that you should be able to see your own account information, assuming you have already been successfully authenticated into the application. After logging into The bAInk web portal and asking Aiya for your account information, you should be able to see the relevant and valid information being returned.

Add FGA Client

  1. Open lib/auth0/fga/client.ts in your code editor
  2. Create (internal) FGA client.
  1. You will need to pull in the .env variables you set up previously to initialize a new instance of OpenFgaClient
  2. You will need, at a minimum, the following options:

{

    apiUrl: process.env.FGA_API_URL,

    storeId: '',

    credentials: {

      method: CredentialsMethod.ClientCredentials,

      config: {

        apiTokenIssuer: '',

        apiAudience: '',

        clientId: '',

        clientSecret: ''

      }

    }

  }

  1. Export a public getter but this step is already done for you!

[!Note] You should now see Success! FGA client initialized in the console. If so, then great job!

If not…check in with Aiya or a lab attendant to see where things may have gone awry.

You may need to refresh the app.

Add getAccountPermissions Tool

  1. Open lib/auth0/fga/get-account-permissions.ts in the code editor
  2. Define the relations to check for

[

 "can_view",

 "can_view_balances",

 "can_view_transactions",

 "can_transfer"

]



  1. Build the FGA batch checks.
  1. Build an array of checks to send to FTA in batch.
  2. Think of it as asking FGA Does user:<customerId> have <relation> on account:<id>? You just need to ask in a way FGA understands!
  1. Make a batch request to FGA via the client SDK but this step is already done for you!
  2. Build/collect allowed relations per account id but this step is already done for you as well!
  3. Build the final accounts response
  1. Don’t over think this one! It seems complicated but it is simple:
  1. What permissions did FGA return for this account?
  2. How should the response be modified, if at all, to adjust for what FGA returned?
  3. Return the account data
  1. Return the modified response but this step is already done for you!

Pulse Check

  1. Open the ‘dev tools’ menu (right-side of app) and click on Reset Account Permissions.
  2. You should now be successfully seeing account information. If you do see account information, but it is not complete then you may not be checking for permissions correctly.
