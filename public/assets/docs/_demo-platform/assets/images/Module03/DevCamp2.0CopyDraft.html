Lab 3 Objective

Copy the correct Auth0 tenant server-side variables into our local environment variable file.  Enable Auth0’s management API through the Auth0 management dashboard. Accomplishing the prior tasks will allow us to successfully verify user authentication.

Description

Now that we have successfully set up the application and it’s running in our local environment, let’s go ahead and start configuring the correct variables to ensure Aiya is able to properly respond to user prompts. We’ll pull these variable values directly from our Auth0 tenant through the dashboard. Subsequently, we’ll also dive into the dashboard to ensure that Aiya has access to the Auth0 management API in order to communicate directly with Auth0 to manage tenant resources. Finally, we will conclude by verifying that our application is set up properly to handle user authentication and registration.

Add Auth0 Tenant .env server-side variables

  1. Navigate to the root project folder where you cloned the repo (./auth0-ai) and open the .env file.
  2. The following values are already in the .env file but commented out. Either update their values (and uncomment them) or simply add the following as-is:
  1. AUTH0_DOMAIN={{idp.tenantDomain}}
  2. AUTH0_CLIENT_ID={{TheBAInk.credentials.clientId}}
  3. AUTH0_CLIENT_SECRET={{TheBAInk.credentials.clientSecret}}

[!important] My values do not seem to be pre-populated. Where else can I find them? The values above should be prepopulated. If you are seeing something like {{idp.tenantDomain}} instead of a url, follow these steps:

  1. Navigate to the Auth0 Demo Platform Dashboard
  2. Select your tenant (the one created for this lab – you probably already have it open!).
  3. Go to Applications -> Applications -> [the bAInk]
  4. Copy the Domain, Client ID, and Client Secret

Add Auth0/NextJS .env secret

Next, we need to generate a secret that the Auth0 NextJS SDK uses to encrypt the session and transaction cookies.

  1. In your terminal copy and run the following command:
  1. openssl rand -base64 32
  1. Copy the resulting value and add it as the AUTH0_SECRET variable’s value
  1. AUTH0_SECRET=<randomly-generated-value>

Save & Restart

Just to be safe, restart your dev server so it picks up the new environment variables:

  1. Terminate the app:
  1. ^ + C (mac)
  2. Ctrl + C (PC)
  1. Start the app again. If the app does not auto-refresh, make sure to manually refresh it
  1. Npm run dev

Enable Management API

This application utilizes the management API to fetch/update profile information as well as manage authenticators for MFA.

Before it can be used, it needs to be enabled.

  1. Return to the Auth0 Admin Dashboard
  2. Navigate to Applications -> the bAInk -> APIs
  3. On the Auth0 Management API enabled Authorized
  1. ![Enable Management API](./assets/images/the-bAInk-mgmt-enable.png)
  1. Click the arrow next to the toggle to expand the menu
  2. Click to select All
  3. Click Update
  1. ![Select All Management API Scopes](./assets/images/the-bAInk-mgmt-scopes.png)
  1. On the warning that appears, click Continue
  1. ![Select All Management API Scopes](./assets/images/the-bAInk-mgmt-warn.png)

Test Authentication

  1. Return to the open app or reopen at http://localhost:3000
  2. Click Sign Up
  1. [!NOTE] If you do not see Sign up you can always access sign up from Log In as well.
  2. [!TIP] For an even faster experience, try using Continue with Google
  1. Create a user. Use any email you want but we recommend you actually have access to it.
  2. Pick a password
